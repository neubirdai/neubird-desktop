    Falcon Architecture

    A reference for the main entities, data structures, and execution workflows across TUI and HTTP (v1/infer) modes.

    Main Entities

    AgentDefinition

    Declared in internal/agents/agents.go. Describes a named AI persona with its model and capability constraints.

    type AgentDefinition struct {
    Name             string        // "Responder" | "Analyst" | "Architect"
    PersonaName      string        // optional human name for the persona
    Description      string        // one-liner shown in agent picker
    Persona          string        // injected into system prompt
    Model            string        // Claude model ID
    Speed            int           // 1–3 (1 = slow/quality, 3 = fast)
    Quality          int           // 1–3 (1 = fast, 3 = quality)
    MaxTokens        int           // optional output token cap
    Skills           []string      // allowlist of skill IDs (nil = all)
    Tools            []string      // allowlist of tool names (nil = all)
    DisableSynthesis bool          // skip Opus synthesis handoff
    AutoContinue     bool          // loop until fully answered (server/daemon mode)
    Timeout          time.Duration // per-run timeout (0 = default 10 min)
}

    Built-in agents:

    Name Model Speed/Quality Purpose
    Responder Haiku 3/1 Fast triage, bullet-point answers
    Analyst Sonnet 2/2 Methodical RCA, shows reasoning
    Architect Opus 1/3 Runbooks, design docs, long-horizon

    Custom agents are loaded from .md files (YAML frontmatter + persona body) in agents/ or ~/.config/neubird/agents/.

    Registry safety

    The agents.All slice is protected by allMu sync.RWMutex. The public API provides two safe accessors:

    • agents.Get(name) — returns a heap-allocated copy of the matching entry. The pointer remains valid even if Init() later reallocates the backing array.
    • agents.Snapshot() — returns a full copy of the registry slice under a read lock. Use this for iteration (e.g., agent picker, /agent command). Do not range over agents.All directly.

    Session

    Declared in internal/ai/interfaces.go. The stable container for a conversation thread.

    type Session struct {
    ID        string       // stable UUID; flows through all traces
    Messages  safeMessages // thread-safe, append-only message list
    CreatedAt time.Time
}

    A Session accumulates all turns (user, assistant, tool calls, tool results). It is created once and reused across multiple Execute() calls to maintain conversation context. In server mode, sessions are pooled by "sessionID|pgEndpoint" key with a 30-minute TTL.

    Transport

    Declared in internal/ai/interfaces.go. Handles the LLM wire protocol.

    type Transport interface {
    Ping(ctx context.Context) error
    SendMessageStream(ctx, systemPrompt, messages, tools, outputSchema, cb) (*APIResponse, error)
    SendSynthesisStream(ctx, systemPrompt, messages, outputSchema, cb) (*APIResponse, error)
    SendDigestRequest(ctx, systemPrompt, messages) (*APIResponse, error)
}

    There are two distinct transport lifecycles:

    1. Per-Execute transport (rs.transport): created fresh at the start of each Execute call for the main LLM loop, and closed in a deferred cleanup when Execute returns. Carries the agent model, debug log pointer, and other per-call state. Fully isolated — concurrent Execute calls never share this transport.

    2. Digest transport (a.digestTransport): created once in NewAgent and reused for all background interim-summary (Haiku digest) calls across the agent's lifetime. Closed in Close() only after draining all in-flight goroutines via a.interim.wg.Wait(). Using a shared digest transport avoids spawning a new HTTP client (and TLS handshake) for every tool round.

    Agent struct — shared vs per-run state

    agent in internal/ai/agent.go is a long-lived struct. It holds two kinds of state: shared (safe for concurrent Execute calls) and per-run (isolated to a single Execute invocation via runState).

    Shared state (on agent struct)

    These fields are either immutable after NewAgent, or protected by their own synchronisation primitive:

    Field Type Protection Notes
    baseCfg ProviderConfig immutable LLM provider credentials
    basePromptSection *MutableSection sync.RWMutex inside Overridden per-agent; read on every prompt build
    schemaSection *MutableSection sync.RWMutex inside DB schema; updated by SetSchemaContext
    extraSection *MutableSection sync.RWMutex inside Extra instructions
    tableSection *MutableSection sync.RWMutex inside Table learnings
    insightsSection *MutableSection sync.RWMutex inside User insights
    skillsSection *MutableSection sync.RWMutex inside Live shared skills text (used when no per-run allowlist)
    skills skillsRegistry sync.RWMutex inside Map of all loaded local skills
    mcp mcpRegistry sync.RWMutex inside MCP tool list + sessions
    sentinel sentinelManager sync.Mutex + atomic.Pointer Background sentinel lifecycle
    watch watchManager sync.Mutex + atomic.Pointer Background watch lifecycle
    digestTransport Transport WaitGroup drain before close Reused for all Haiku digest calls
    interim interimState sync.Mutex + sync.WaitGroup Background summarizer state
    lastResult safeRunResult sync.RWMutex Outcome of most recent Execute
    ace aceConnector sync.Mutex + atomic.Int64 ACE reconnect deduplication
    store *db.Store — Postgres store; methods are concurrency-safe
    learnings, tableLearnings, feedbackStore store types — Concurrency-safe by design
    agentName atomic.Value atomic Set by TUI, read by agent goroutine

    Per-run state (on runState)

    runState is created fresh at the top of each Execute call and never shared with other concurrent calls. This is the key mechanism that makes it safe to call Execute concurrently on the same agent.

    type runState struct {
    // Infrastructure (resolved from AgentDefinition + RunConfig)
    transport        Transport        // fresh client for this model/call
    buildPrompt      func() string    // closure combining per-run + shared sections
    allowedTools     map[string]bool  // nil = all tools
    allowedSkills    map[string]bool  // nil = all skills
    disableSynthesis bool
    autoContinue     bool

    // Execution tracking
    session          *Session
    sources          []SourceEntry    // data sources touched this run
    progress         *ProgressTracker
    turns            []TurnSnapshot   // per-turn reasoning chain for ACE
    toolLog          *ToolLog
    actionsMu        sync.Mutex
    summaryMu        sync.Mutex
    turnSummaries    map[int]string   // interim summaries keyed by turn (async)

    // Chart accumulation (per-run; never shared with concurrent calls)
    charts   []ChartEntry
    chartsMu sync.Mutex

    // Other per-call fields (counters, timestamps, question text, etc.)
    ...
}

    Prompt sections and lazySection

    internal/ai/prompt_sections.go defines the PromptSection interface:

    type PromptSection interface {
    Build() string
}

    Implementations:

    Type When to use
    StaticSection Immutable text (persona, agent prompt)
    MutableSection Shared text updated at runtime; thread-safe via sync.RWMutex
    DateSection Always returns current UTC date/time
    lazySection Re-evaluates a closure on every Build() call

    lazySection is used for the per-run skills prompt when a skills allowlist or autoContinue mode is active. Because InjectLocalSkills can add skills mid-run, the closure captures a reference to skillsRegistry.snapshot() (called on each Build()), so new skills injected between LLM turns are automatically reflected in the next turn's system prompt — without any lock coordination between the Execute goroutine and the skills injector.

    When neither an allowlist nor autoContinue is set, the run simply reuses the engine-level a.skillsSection (MutableSection) directly.

    InfraConfig

    Declared in internal/ai/agent.go. Wires the agent to external infrastructure.

    type InfraConfig struct {
    MCPConfigPath       string     // path to MCP JSON config
    LocalTools          bool       // enable run_local_command
    LocalToolsEnableGit bool
    LocalToolsEnableAz  bool
    Ace                 *AceConfig // cloud-native MCP mode
}

    RunConfig

    Declared in internal/ai/interfaces.go. Per-call overrides applied on top of AgentDefinition.

    type RunConfig struct {
    MaxIterations    int
    FreshHistory     bool
    AutoContinue     bool          // merged with AgentDefinition.AutoContinue (OR)
    AllowedTools     []string
    AllowedSkills    []string
    DisableSynthesis *bool         // nil = use AgentDefinition.DisableSynthesis
    SessionID        string
    PromptID         string
    OutputSchema     json.RawMessage
    SuppressPersist  bool
}

    Resolution order for key fields:

    • autoContinue = runCfg.AutoContinue || agentDef.AutoContinue
    • disableSynthesis: runCfg.DisableSynthesis (if non-nil) → agentDef.DisableSynthesis
    • timeout: agentDef.Timeout → TIMEOUT_SECONDS env var → 10 minutes

    Skill

    Declared in internal/skills/loader.go. A named, injectable prompt fragment with optional reference files.

    type Skill struct {
    Name                   string
    Desc                   string
    Prompt                 string            // markdown content
    Files                  map[string]string // path → content
    Tags                   []string
    UserInvocable          bool
    DisableModelInvocation bool
    EnableStructuredOutput bool
    StructuredOutputSchema json.RawMessage
}

    Skills are loaded from (in priority order, last wins):

    1. Embedded built-ins (compiled into binary)
    2. ./skills/ in current working directory
    3. ~/.config/neubird/skills/
    4. $HAWKEYE_SKILLS_DIR

    MCP Integration

    Declared in internal/ai/tools_mcp.go. External tool servers via Model Context Protocol.

    type MCPServerConfig struct {
    Command   string
    Args      []string
    Env       map[string]string
    URL       string            // HTTP endpoint; if set → http transport
    Headers   map[string]string
    Timeout   time.Duration
}
    • Config read from ~/.config/neubird/mcp_config.json (or $NEUBIRD_MCP_CONFIG).
    • On agent init, MCP servers are spawned/connected and tools/list is called to register tools.
    • A background goroutine reconnects on failure.
    • Tools hidden from LLM via _meta.x_hidden_from_llm.

    Background interim summarizer

    After each tool-use round, launchBackgroundSummarizer (in internal/ai/interim_summary.go) spins up a goroutine that calls digestTransport.SendDigestRequest with a snapshot copy of conversation history (never a.history directly). It produces a short "what we know so far" paragraph emitted via OnInterimSummary.

    Concurrency invariants:

    • Only one background goroutine runs at a time (interimState.bgRunning flag, protected by interimState.mu).
    • Rate-limited to one launch per 10 seconds.
    • wg.Add(1) is called inside the mutex (when bgRunning is set), so Close() calling wg.Wait() either blocks until the goroutine finishes or sees count=0 before any goroutine was committed — there is no window where wg.Wait() can return while a goroutine is about to start.
    • If the delta is empty (nothing new since the last summary), bgRunning is cleared and wg.Done() is called immediately — no goroutine is launched but the wg.Add is always balanced.
    • Close() calls a.interim.wg.Wait() before closing digestTransport, preventing use-after-close races.
    • Stale results from a previous investigation are detected via interimState.generation (incremented by resetInterimState).

    Workflows

    TUI Mode

    Startup

    main()
  └─ tui.New(cfg)
       ├─ agents.Init()                   → merges disk agents into agents.All (once, under allMu.Lock)
       ├─ skills.LoadMerged(...)          → []Skill
       ├─ ai.NewAgent(infraCfg, store)
       │    ├─ NewClient(baseCfg)         → digestTransport (shared lifetime)
       │    ├─ loadMCPTools()             → registers MCP sessions + tools
       │    └─ registerLocalTools()
       ├─ ai.NewSession()
       └─ model.Init()
            ├─ Ping LLM (validate API key)
            ├─ db.Connect(pgEndpoint)
            └─ agent.SetSchemaContext()   → discovers tables/columns

    Per-Question Flow

    User types prompt + Enter
  └─ submitInput()
       └─ agent.Run(ctx, session, activeAgent, prompt, AgentCallback{...})
            │  [runs in goroutine]
            │
            ├─ executeInternal():
            │   ├─ Create rs.transport (fresh per-call, closed on return)
            │   ├─ Resolve rs.allowedTools, rs.allowedSkills, rs.disableSynthesis, rs.autoContinue
            │   ├─ Build rs.buildPrompt closure (lazySection for skills if allowlist/autoContinue active)
            │   │
            │   ├─ Tool loop:
            │   │   ├─ rs.transport.SendMessageStream(rs.buildPrompt(), messages, tools)
            │   │   │   ├─ OnText    → callback.OnText()
            │   │   │   └─ tool_use → execute → callback.OnToolExec + OnToolResult
            │   │   ├─ Append messages to session
            │   │   ├─ launchBackgroundSummarizer(historySnapshot) [non-blocking]
            │   │   └─ Repeat until end_turn or max_tokens
            │   │
            │   └─ Synthesis (if stop_reason=max_tokens and !rs.disableSynthesis)
            │       └─ rs.transport.SendSynthesisStream(Opus) → refined answer
            │
            └─ OnDone / OnError

Bubble Tea message loop
  └─ model.View() reads StreamProcessor → renders ANSI to terminal

    Agent Switching

    User selects agent from picker (Ctrl+A or /agent)
  └─ applyAgent(def)
       └─ m.activeAgent = &agCopy   (heap copy via agents.Get or Snapshot()[i])
          (rs.transport is created fresh on each Execute call — no manual recreation needed)

    The agent picker and /agent command iterate agents.Snapshot() (not agents.All directly) so the reads are formally race-free even if agents.Init() reallocates the backing array.

    Skills

    Slash-commands (e.g., /sev-hunt) inject the skill's prompt template as a prefix to the next user message.

    HTTP Server Mode (POST /v1/infer)

    Request

    {
  "prompt": "Why is p99 latency spiking?",
  "session_id": "abc-123",
  "pg_endpoint": "postgres://...",
  "agent": "Analyst",
  "skills": ["sev-hunt"],
  "tools": ["exec_sql"]
}

    Flow

    POST /v1/infer
  └─ handleInfer()
       ├─ buildPGEndpoint()             → resolve connection string
       ├─ Set SSE headers (text/event-stream)
       ├─ getStore(pgEndpoint)          → pooled db.Store (cached by endpoint)
       │
       ├─ getSession(sessionID|pgEndpoint)   → *sessionEntry from sync.Map
       │    ├─ [cache hit]  reuse Agent + Session
       │    └─ [cache miss] ai.NewAgent + agent.SetSchemaContext()
       │
       ├─ Resolve AgentDefinition + RunConfig from request fields
       │
       └─ jobrunner.Execute(ctx, agent, session, spec, writer.SSE(w))
            ├─ agent.Execute(ctx, session, def, prompt, callback)
            │   [same tool loop as TUI]
            │
            └─ callback → writer.WriteEvent(event) → Flush()
                  SSE frames: {"type":"text_chunk","text":"..."}
                              {"type":"tool_start",...}
                              {"type":"tool_result",...}
                              {"type":"done",...}

    Database Modes

    Mode Trigger How
    Direct pg_endpoint in request Connect directly to Postgres
    Project project_uuid + RAVEN_STORE_DB_URL Raven resolves endpoint
    ACE No Postgres; aceConfig set MCP provides all tools

    Session Pool

    sync.Map key: "sessionID|pgEndpoint"
Value: sessionEntry {
    agent     ai.Agent    ← reused (holds MCP connections, schema cache, digestTransport)
    session   *ai.Session ← reused (holds message history)
    lastUsed  time.Time
}
Background eviction: every 2 min, remove entries idle > 30 min

    Agent Execution Engine (shared by TUI + Server)

    agent.Execute() in internal/ai/agent.go:

    Timeout contract: Execute owns its timeout budget entirely. It strips the parent deadline via context.WithoutCancel, then sets a fresh deadline from def.Timeout (falling back to TIMEOUT_SECONDS env var, then 10 minutes). context.AfterFunc propagates genuine user cancellations (Ctrl+C / HTTP disconnect) without inheriting the caller's deadline. Do not pass a deadline-bearing context — the deadline cannot extend def.Timeout, and if it fires first it is indistinguishable from a user cancellation, causing the run to be misclassified as CANCELLED instead of FAILED. Set def.Timeout instead. Callers that wrap Execute (Sentinel, Watch, investigate CLI, connect) set def.Timeout rather than wrapping the context in WithTimeout.

    Execute(ctx, session, agentDef, prompt, callback, runCfg)
  │
  ├─ 1. Create rs (runState) — fresh per-call, never shared
  │       rs.transport = NewClient(agentDef.Model)   [closed on return]
  │       rs.autoContinue = runCfg.AutoContinue || agentDef.AutoContinue
  │       rs.disableSynthesis = (resolved from runCfg + agentDef)
  │       rs.allowedTools, rs.allowedSkills = (resolved from runCfg + agentDef)
  │
  ├─ 2. Build rs.buildPrompt closure
  │       If allowedSkills != nil or autoContinue:
  │         → lazySection (re-evaluates skills on each Build() call)
  │       Else:
  │         → reuse a.skillsSection (shared MutableSection)
  │       Combined with: persona + base prompt + DateSection + schema +
  │                      table learnings + extra sections + insights
  │
  ├─ 3. Resolve allowed tools
  │       runCfg.AllowedTools → agentDef.Tools → nil (all tools)
  │
  ├─ 4. Create OTel span (session ID, user, org, project, MCP tools)
  │
  ├─ 5. Tool loop
  │       a. rs.transport.SendMessageStream(rs.buildPrompt(), messages, tools)
  │       b. Parse content blocks:
  │           - text → callback.OnText()
  │           - tool_use → execute tool → callback.OnToolExec + OnToolResult
  │       c. Append messages to session
  │       d. launchBackgroundSummarizer(historySnapshot)   [non-blocking]
  │       e. If stop_reason=tool_use → repeat from (a)
  │       f. If stop_reason=end_turn → done
  │       g. If stop_reason=max_tokens + !rs.disableSynthesis → synthesis
  │
  └─ 6. Synthesis (optional)
          rs.transport.SendSynthesisStream() with Opus
          Opus synthesizes refined answer from full context
          callback.OnText(synthesized) → callback.OnDone()

    Agent Lifecycle

    NewAgent(infraCfg, store)
  ├─ digestTransport = NewClient("")   [lifetime: until Close()]
  ├─ loadMCPTools()
  └─ registerLocalTools()

Execute(ctx, session, def, prompt, cb, cfg...)  [may be called concurrently]
  ├─ Creates fresh runState (transport + prompt closure + per-run config)
  ├─ Runs tool loop
  ├─ May launch background summarizer goroutines (tracked by interim.wg)
  └─ Closes rs.transport on return

Close()
  ├─ interim.wg.Wait()             ← drain in-flight digest goroutines first
  ├─ digestTransport.Close()       ← safe: no goroutine can use it after wg drains
  ├─ sentinel.stop()
  ├─ watch.stop()
  └─ mcp.closeAll()                ← closes MCP server connections
