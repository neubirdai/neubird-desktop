    SSO Login with Device Flow

    This document describes the OAuth 2.0 Device Authorization Flow implementation for Falcon CLI SSO authentication.

    Overview

    Falcon uses OAuth 2.0 Device Authorization Flow to authenticate with Raven through Auth0 SSO. This flow is specifically designed for CLI applications and supports enterprise SSO providers like:

    • Ping Identity (SAML/OIDC)
    • Okta
    • Azure Entra ID (formerly Azure AD)
    • Google Workspace
    • Any other Auth0 enterprise connection

    Why Device Flow?

    Device Flow is the industry standard for CLI authentication because:

    ✅ No localhost server required - No port conflicts or firewall issues
    ✅ Works in any environment - SSH, containers, headless systems
    ✅ SSO compatible - Seamlessly integrates with enterprise identity providers
    ✅ Secure - No credentials stored in CLI, tokens never exposed in URLs
    ✅ User-friendly - Simple browser-based authentication
    ✅ Widely adopted - Used by AWS CLI, Azure CLI, GitHub CLI, etc.

    Quick Start

    Prerequisites

    Set these environment variables:

    export AUTH0_DOMAIN="neubird.us.auth0.com"          # Your Auth0 tenant domain
export AUTH0_ORG_ID="73711cf7-89c7-44d1-9fce-xxx"   # Your organization UUID
export AUTH0_AUDIENCE="neubird-api"                  # Optional, defaults to "neubird-api"
export OAUTH_CLIENT_ID="falcon-cli"                  # Optional, defaults to "falcon-cli"

    Login

    neubird login https://myenv.app.neubird.ai --sso

    What happens:

    1. CLI requests a device code from Auth0
    2. You see a URL and a user code (e.g., ABCD-EFGH)
    3. Browser opens automatically (or you can open manually)
    4. Enter the code in your browser
    5. Authenticate via your corporate SSO
    6. Return to terminal - login complete!

    Example output:

   Auth0 Domain: neubird.us.auth0.com

   1. Visit: https://neubird.us.auth0.com/activate?user_code=ABCD-EFGH

   2. Or visit: https://neubird.us.auth0.com/activate
      and enter code: ABCD-EFGH

✓ Login successful!

    Configuration

    Environment Variables

    Variable Required Default Description
    AUTH0_DOMAIN No neubird.us.auth0.com Auth0 tenant domain
    AUTH0_ORG_ID Yes - Organization UUID for SSO routing
    AUTH0_AUDIENCE No neubird-api Auth0 API audience
    OAUTH_CLIENT_ID No falcon-cli OAuth client ID

    Finding Your Organization ID

    Your organization ID is the UUID that identifies your organization in Raven. You can find it:

    1. In the Raven web UI (Organization Settings)
    2. From your administrator
    3. In the URL when logged into Raven: https://xxx.app.neubird.ai/org/{org-uuid}/...

    How It Works

    Device Flow Sequence

    ┌─────────┐                 ┌──────────┐                 ┌─────────┐
│  Falcon │                 │  Browser │                 │  Auth0  │
│   CLI   │                 │          │                 │         │
└────┬────┘                 └────┬─────┘                 └────┬────┘
     │                           │                            │
     │ 1. POST /oauth/device/code                            │
     │    + organization                                      │
     ├───────────────────────────────────────────────────────>│
     │                           │                            │
     │ 2. device_code, user_code │                            │
     │<───────────────────────────────────────────────────────┤
     │                           │                            │
     │ 3. Display URL + code     │                            │
     │                           │                            │
     │ 4. Open browser           │                            │
     ├──────────────────────────>│                            │
     │                           │                            │
     │                           │ 5. Visit activation URL    │
     │                           ├───────────────────────────>│
     │                           │                            │
     │                           │ 6. Enter user_code         │
     │                           ├───────────────────────────>│
     │                           │                            │
     │                           │ 7. Redirect to SSO         │
     │                           │<───────────────────────────┤
     │                           │                            │
     │                           │ 8. Authenticate with SSO   │
     │                           │    (Ping/Okta/Azure)       │
     │                           │                            │
     │                           │ 9. Grant permission        │
     │                           ├───────────────────────────>│
     │                           │                            │
     │ 10. Poll POST /oauth/token                             │
     │     + device_code                                      │
     ├───────────────────────────────────────────────────────>│
     │                           │                            │
     │ 11. authorization_pending │                            │
     │<───────────────────────────────────────────────────────┤
     │                           │                            │
     │ 12. Poll again (every 5s) │                            │
     ├───────────────────────────────────────────────────────>│
     │                           │                            │
     │ 13. access_token          │                            │
     │     + refresh_token       │                            │
     │<───────────────────────────────────────────────────────┤
     │                           │                            │
     │ 14. Save to ~/.config/neubird/neubird.json             │
     │                           │                            │

    Key Components

    1. Device Code Request (internal/oauth/device_flow.go)
    2. Requests device code from Auth0
    3. Includes organization parameter for SSO routing

    4. Returns user code and verification URL

    5. User Authentication (Browser)

    6. User visits verification URL
    7. Enters user code
    8. Auth0 detects organization and redirects to SSO provider

    9. User authenticates via corporate SSO

    10. Token Polling (internal/oauth/device_flow.go)

    11. CLI polls Auth0 every 5 seconds
    12. Handles authorization_pending, slow_down, etc.

    13. Receives tokens when user completes authentication

    14. Credential Storage (internal/config/credentials.go)

    15. Saves access token, refresh token, expiration
    16. Stored in ~/.config/neubird/neubird.json
    17. File permissions set to 0600 (owner read/write only)

    Auth0 Configuration

    Application Settings

    Your Auth0 application must be configured as follows:

    Application Type: Native (or Single Page Application)

    Grant Types:

    • ✅ Device Code (required)
    • ✅ Refresh Token (recommended)

    Token Endpoint Authentication: None (public client)

    Organization Settings:

    • Organization must be configured in Auth0
    • Enterprise connection (Ping, Okta, Azure) enabled for the organization
    • Organization "name" field should match Raven org UUID

    API Permissions

    The application needs these scopes:

    • openid - Basic authentication
    • profile - User profile information
    • email - User email address
    • offline_access - Refresh tokens for long-lived sessions

    Token Management

    Token Storage

    Tokens are stored in ~/.config/neubird/neubird.json:

    {
  "raven_url": "https://bao.app.neubird.ai",
  "access_token": "eyJhbGc...",
  "refresh_token": "v1.MRr...",
  "expires_at": 1742486255,
  "org_uuid": "73711cf7-89c7-44d1-9fce-xxxxxxxxxxxx",
  "profile": "default"
}

    Token Expiration

    • Access tokens typically expire after 24 hours
    • Refresh tokens are long-lived (can be used to get new access tokens)
    • Falcon automatically refreshes expired tokens when needed

    Token Refresh

    The Device Flow client includes automatic token refresh:

    client := oauth.NewDeviceFlowClient(auth0Domain, "", "")
newToken, err := client.RefreshToken(ctx, refreshToken)

    Troubleshooting

    "AUTH0_ORG_ID environment variable is required"

    Problem: Organization ID not configured.

    Solution:

    export AUTH0_ORG_ID="your-org-uuid-here"

    Get your org UUID from your administrator or Raven web UI.

    "device code request failed"

    Problem: Auth0 configuration issue or network error.

    Solution:

    1. Verify AUTH0_DOMAIN is correct
    2. Check network connectivity to Auth0
    3. Verify Auth0 application has Device Flow enabled
    4. Check Auth0 application logs for errors

    "authentication timeout"

    Problem: User didn't complete authentication within the time limit (usually 15 minutes).

    Solution:

    • Run the login command again
    • Complete authentication more quickly
    • Check if your SSO provider is responding

    "user denied authorization"

    Problem: User clicked "Deny" in the browser.

    Solution:

    • Run the login command again
    • Click "Allow" when prompted

    "authentication failed: invalid_grant"

    Problem: Device code expired or already used.

    Solution:

    • Run the login command again to get a new device code

    Browser doesn't open automatically

    Problem: Browser opening failed (common in SSH sessions).

    Solution:

    • Copy the URL manually from the terminal
    • Paste it into your browser
    • Enter the user code when prompted

    Security Considerations

    ✅ No credentials in CLI - Username/password never stored
    ✅ Tokens encrypted at rest - File permissions 0600
    ✅ Short-lived access tokens - Expire after 24 hours
    ✅ Refresh token rotation - Can be configured in Auth0
    ✅ Organization isolation - SSO routing via organization parameter
    ✅ Audit logging - All authentications logged in Auth0

    Comparison with Authorization Code Flow

    Feature Device Flow Authorization Code Flow
    Localhost server ❌ Not required ✅ Required (port 8080-8099)
    SSH/headless ✅ Works ❌ Difficult
    Firewall friendly ✅ Yes ⚠️ May block localhost
    User experience Manual code entry Automatic redirect
    Security ✅ Excellent ✅ Excellent (with PKCE)
    Industry adoption ✅ AWS, Azure, GitHub CLI ⚠️ Less common for CLI

    Examples

    Basic Login

    export AUTH0_DOMAIN="neubird.us.auth0.com"
export AUTH0_ORG_ID="73711cf7-89c7-44d1-9fce-xxxxxxxxxxxx"

neubird login https://myenv.app.neubird.ai --sso

    Custom Auth0 Configuration

    export AUTH0_DOMAIN="custom-tenant.auth0.com"
export AUTH0_ORG_ID="my-org-uuid"
export AUTH0_AUDIENCE="custom-api"
export OAUTH_CLIENT_ID="my-client-id"

neubird login https://myenv.app.neubird.ai --sso

    SSH/Headless Environment

    # In SSH session where browser can't open
neubird login https://myenv.app.neubird.ai --sso

# Copy the URL from terminal output
# Open it in your local browser
# Enter the code shown in terminal
# Return to SSH session - authentication complete!
